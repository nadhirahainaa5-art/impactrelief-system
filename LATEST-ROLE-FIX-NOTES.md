# Latest Role Fix Notes

This update aligns the project with the locked Option A donor model.

## Final separation applied
- Donor self-registers and manages only own profile and donations.
- Staff handles donation operations and verification only.
- Admin handles governance, campaigns, fund allocations, approvals, audit visibility, and system-level oversight.

## Fixed in this update
- Staff can no longer access fund allocation management.
- Staff can no longer access purpose management.
- Admin is no longer the main daily operator for donation or expense entry.
- Staff can now verify or fail donations.
- Staff can create, edit, and submit expenses only for their own records.
- Admin can approve or reject expenses.
- Campaigns remain viewable to all logged-in roles, but only admin can create, edit, or delete them.
- Navbar was cleaned so each role only sees the correct professional one-line menu.
